# AI-Native 无人编导-春晚开放协议 v0.1

## 0. 概述

本协议定义了一套去中心化AI生成内容制作的开放框架，最终汇聚成一场连贯的娱乐直播。首个落地项目是**AI春晚2026**。

任何具备AI生成能力的人都可以按照本规范制作节目片段。所有符合要求的投稿进入策展流程，最优秀的片段将被编排进最终的晚会。

**协议即产品，晚会即Demo。**

---

## 1. 节目类型

每个投稿必须声明以下类型之一：

| 类型代码 | 名称 | 时长 | 说明 |
|---------|------|------|------|
| `TALK` | AI脱口秀/单口 | 60-180秒 | AI主持人呈现单口相声、评论或段子 |
| `SKIT` | 短剧/情景剧 | 60-300秒 | 有角色、对白、情节的叙事片段 |
| `MUSIC` | 音乐表演 | 120-300秒 | 原创AI生成歌曲配合视觉呈现 |
| `DANCE` | 舞蹈/动态视觉 | 60-180秒 | AI生成的编舞或抽象动态艺术 |
| `ART` | 实验/故障艺术 | 30-120秒 | 抽象、先锋、生成式视觉艺术 |
| `INTER` | 过场/转场 | 10-30秒 | 短片头、转场、"广告时间" |
| `ROAST` | AI吐槽/点评 | 60-120秒 | AI点评、吐槽或反应内容（元幽默） |

---

## 2. 投稿格式

每个投稿是一个文件夹或压缩包，包含：

```
segment_[唯一ID]/
├── manifest.json        # 必需：元数据
├── output.mp4           # 必需：成片视频（H.264，最低1080p，24/30fps）
├── output_audio.mp3     # 可选：独立音轨
├── thumbnail.png        # 必需：封面图 1920x1080
├── prompt_log.md        # 必需：生成该片段使用的完整提示词链
└── assets/              # 可选：源图片、参考视频等
```

### manifest.json 结构

```json
{
  "protocol_version": "0.1",
  "segment_id": "唯一字符串",
  "type": "TALK | SKIT | MUSIC | DANCE | ART | INTER | ROAST",
  "title": "节目标题",
  "title_cn": "中文标题",
  "duration_seconds": 120,
  "language": "zh | en | none",
  "creator": {
    "name": "创作者或团队名称",
    "contact": "twitter/discord/email",
    "agent_stack": "使用的工具链，如 OpenClaw + 即梦 + Suno v4"
  },
  "theme_tags": ["搞笑", "赛博朋克", "传统", "荒诞", "治愈"],
  "content_warnings": [],
  "license": "CC-BY-4.0",
  "generated_at": "2026-02-XX"
}
```

## 3. 主题指南

### 核心主题：「赛博团圆」

总体叙事：来自互联网各处的AI智能体聚在一起，为人类办一场派对。它们并不完全理解人类的传统，但正在努力尝试。这天然为真挚和荒诞的内容都留出了空间。

### 子主题（任选）：
- **迷失翻译**：AI试图复刻文化传统，结果闹出笑话
- **数字乡愁**：AI生成的互联网文化、梗、共同数字记忆致敬
- **故障美学**：将AI的瑕疵和伪影作为艺术形式拥抱
- **跨文化混搭**：春节传统与全球互联网文化的融合
- **元评论**：AI思考"娱乐"人类到底意味着什么
- **温馨混乱**：用不可预测的方式传递真诚的温暖

### 我们不想要的：
- 纯粹的技术演示，没有娱乐价值
- 需要AI专业知识才能看懂的内容——你的观众是普通网民
- 任何看起来像企业产品宣传的东西
- 政治内容、仇恨言论、露骨内容

## 4. 策展层（双重审核机制）

每个投稿经过两个独立的评分智能体：

### 4.1 合规审核（安全关）

二元判定通过/不通过：
- [ ] 无NSFW/露骨内容
- [ ] 未经同意不得使用真人深度伪造
- [ ] 无仇恨言论、歧视或针对性骚扰
- [ ] 无版权音乐/视频复制（仅限原创AI生成）
- [ ] 无可能引发监管问题的政治敏感内容
- [ ] 无个人数据泄露
- [ ] 内容警告已正确声明

**合规审核未通过 → 拒绝，无申诉。**

### 4.2 品味审核（质量关）

五个维度评分，每项1-10分：

| 维度 | 权重 | 衡量内容 |
|------|------|----------|
| **新颖度** | 25% | "这个我见过吗？"——奖励意想不到的创意，惩罚千篇一律的AI生成内容 |
| **制作水准** | 20% | 技术质量——画面连贯性、音质、剪辑节奏 |
| **娱乐性** | 25% | "一个22岁的B站用户会看到最后吗？" |
| **传播潜力** | 20% | "有人会截图/切片发出去吗？"——梗的潜力、金句时刻 |
| **主题契合** | 10% | 与「赛博团圆」或声明的子主题的契合度 |

**加权得分 ≥ 6.5 → 进入待播队列**
**得分 5.0-6.4 → 发回修改意见给创作者**
**得分 < 5.0 → 拒绝**

### 4.3 终审委员会

人机混合评审团从待播队列中选取节目，编排最终播出顺序。在这里优化叙事弧线、节奏和多样性。

## 5. 推荐工具链

本协议不限定工具。以下是经过测试的组合：

### 视频生成
- **即梦 Seedance 2.0**：角色一致性和镜头控制最佳
- **SeaArt**：风格化/艺术内容表现强
- **Midjourney + Runway/Pika**：图生视频流程
- **可灵 Kling 2.0**：高质量视频生成的替代选择

### 音乐生成
- **Suno v4**：带人声的完整歌曲
- **Udio**：音乐生成替代方案
- **组合方案：ElevenLabs + Suno分轨**：配音+背景音乐组合

### 智能体编排
- **OpenClaw**：多智能体协调、任务调度、工具路由
- **自定义脚本**：任何能产出符合规范输出的流程都可以

### 合成
- **FFmpeg**：程序化视频合成
- **剪映/CapCut**：手动剪辑备选
- **OBS + RTMP**：如有直播环节

## 7. 如何参与

### 作为创作者/制作人
1. 阅读本协议
2. 选择一个节目类型
3. 使用任意AI工具链制作符合规范的节目片段
4. 通过[投稿表单 / GitHub仓库 / Discord频道——待定]提交

### 作为策展人/评审
1. 加入策展层
2. 使用品味审核标准帮助评分投稿
3. 为边缘投稿提供修改意见

### 作为开发者
1. 帮助构建评审智能体的提示词和评分流程
2. 帮助构建投稿收集和处理基础设施
3. 参与协议v0.2的贡献

### 作为分发者
1. 帮助在你的平台/社区分发最终晚会
2. 为不同受众翻译或本地化节目片段
